# 任务二安全启动指南

## 问题背景
任务二（二阶alpha模拟）需要长时间运行（预计3.2小时），容易因系统空闲检测、网络中断、内存不足等原因被终止。

## 解决方案
我已创建了一套完整的解决方案，确保任务二能够安全、稳定地运行直到完成。

## 创建的组件

### 1. 任务二管理器 (`task2_manager.py`)
- **功能**: 管理任务二的启动、监控、重启
- **特性**: 
  - 进程ID管理
  - 自动重启（最大10次）
  - 进度检查和状态报告
  - 优雅停止

### 2. 防终止监控 (`keep_alive_monitor.py`)
- **功能**: 防止系统因空闲而终止程序
- **特性**:
  - 定期执行系统活动
  - 监控进程健康状态
  - 系统资源检查（内存、磁盘）
  - 状态报告

### 3. 故障恢复管理器 (`recovery_manager.py`)
- **功能**: 处理程序崩溃后的自动恢复
- **特性**:
  - 进度文件验证和修复
  - 自动备份和恢复
  - 崩溃原因分析
  - 自动重启

### 4. 安全启动脚本 (`start_task2_safely.bat`)
- **功能**: 用户友好的启动界面
- **特性**:
  - 多种启动选项
  - 状态检查
  - 一键启动

## 启动方式

### 选项1：完整保护模式（推荐）
```bash
# 双击运行批处理文件
start_task2_safely.bat
```
选择选项1，将同时启动任务二和防终止监控。

### 选项2：手动控制模式
```bash
# 1. 启动任务二
python task2_manager.py start

# 2. 监控任务二状态
python task2_manager.py monitor

# 3. 启动防终止监控（另一个窗口）
python keep_alive_monitor.py

# 4. 查看状态
python task2_manager.py status
python keep_alive_monitor.py status
```

### 选项3：自动恢复模式
```bash
# 启动故障恢复监控（自动处理崩溃）
python recovery_manager.py monitor
```

## 故障处理

### 如果程序停止
1. 检查是否有错误日志
2. 运行故障恢复：
   ```bash
   python recovery_manager.py recover
   ```
3. 重新启动：
   ```bash
   python task2_manager.py restart
   ```

### 如果进度不更新
1. 检查网络连接
2. 检查API服务状态
3. 查看`task2_output.log`中的错误信息
4. 运行崩溃分析：
   ```bash
   python recovery_manager.py analyze
   ```

## 系统要求

- Python 3.6+
- 必需包：`psutil`（已包含自动安装）
- 磁盘空间：至少1GB可用
- 内存：至少2GB可用

## 注意事项

1. **不要关闭运行窗口**：防终止监控需要持续运行
2. **保持网络连接**：程序需要访问API服务
3. **避免同时运行多个实例**：可能导致进度冲突
4. **定期检查状态**：确保程序正常运行

## 紧急停止

如果需要停止程序：
```bash
# 优雅停止
python task2_manager.py stop

# 强制停止（如果上述无效）
taskkill /F /IM python.exe
```
